---
title: "mailcow.conf reference"
description: "Complete reference for every environment variable in mailcow.conf, generated by generate_config.sh."
---

`mailcow.conf` is the main configuration file for your mailcow installation. It is generated by running `generate_config.sh` during initial setup and is read by Docker Compose as the `.env` file (via a symlink).

<Note>
  Edit `mailcow.conf` directly to change settings after the initial setup. Most changes require restarting the affected containers with `docker compose up -d`.
</Note>

---

## Web UI

| Variable | Default | Description |
|---|---|---|
| `MAILCOW_HOSTNAME` | _(prompted)_ | Fully qualified domain name (FQDN) of your mail server. Must contain at least two dots (e.g. `mail.example.com`). The default admin user is `admin` with password `moohoo`. |
| `MAILCOW_PASS_SCHEME` | `BLF-CRYPT` | Password hashing scheme for mailbox passwords. See the [password scheme docs](https://docs.mailcow.email/models/model-passwd/) for all supported values. |

## HTTP/S bindings

<Warning>
  Do not use `IP:PORT` format in `HTTP_BIND`, `HTTPS_BIND`, `HTTP_PORT`, or `HTTPS_PORT`. Do not use ports 8081, 9081, 9082, or 65510.
</Warning>

| Variable | Default | Description |
|---|---|---|
| `HTTP_PORT` | `80` | Host port for HTTP traffic. |
| `HTTP_BIND` | _(empty)_ | IP address to bind HTTP on. Leave empty to bind all interfaces. For a specific IPv4 address use e.g. `1.2.3.4`. |
| `HTTPS_PORT` | `443` | Host port for HTTPS traffic. |
| `HTTPS_BIND` | _(empty)_ | IP address to bind HTTPS on. Leave empty to bind all interfaces. |
| `HTTP_REDIRECT` | `y` | Redirect all HTTP connections to HTTPS (`y`/`n`). |

## Mail ports

<Note>
  These ports are exposed on the host. You should leave them at their standard values unless another service occupies them.
</Note>

| Variable | Default | Description |
|---|---|---|
| `SMTP_PORT` | `25` | SMTP port (inbound and outbound relay). |
| `SMTPS_PORT` | `465` | SMTPS (SMTP over TLS) port. |
| `SUBMISSION_PORT` | `587` | SMTP submission port (STARTTLS). |
| `IMAP_PORT` | `143` | IMAP port (STARTTLS). |
| `IMAPS_PORT` | `993` | IMAPS (IMAP over TLS) port. |
| `POP_PORT` | `110` | POP3 port. |
| `POPS_PORT` | `995` | POP3S (POP3 over TLS) port. |
| `SIEVE_PORT` | `4190` | ManageSieve port for server-side mail filtering rules. |

## Internal ports

These ports are bound to localhost by default and are not exposed to the public internet.

| Variable | Default | Description |
|---|---|---|
| `DOVEADM_PORT` | `127.0.0.1:19991` | Dovecot administration socket port, bound to localhost. |
| `SQL_PORT` | `127.0.0.1:13306` | MariaDB port exposed on the host, bound to localhost. |
| `REDIS_PORT` | `127.0.0.1:7654` | Redis port exposed on the host, bound to localhost. |

## General settings

| Variable | Default | Description |
|---|---|---|
| `TZ` | _(prompted)_ | Server timezone. Use the `TZ identifier` column from the [tz database](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones). |
| `COMPOSE_PROJECT_NAME` | `mailcowdockerized` | Docker Compose project name. Use lowercase letters only. |
| `DOCKER_COMPOSE_VERSION` | _(auto-detected)_ | Switch between `native` (Compose plugin) and `standalone`. At least one must be installed. |
| `ACL_ANYONE` | `disallow` | Set to `allow` to enable the `anyone` pseudo-user in IMAP ACLs, allowing shared folders accessible to all authenticated users. Only enable on single-organisation installations. |
| `MAILDIR_GC_TIME` | `7200` | Time in minutes before deleted mailboxes and domains are permanently removed from `/var/vmail/_garbage`. The garbage collector runs hourly. |

## SSL/ACME

| Variable | Default | Description |
|---|---|---|
| `SKIP_LETS_ENCRYPT` | `n` | Set to `y` to disable the ACME container and use manual or self-signed certificates. |
| `ACME_DNS_CHALLENGE` | `n` | Set to `y` to use the DNS-01 challenge instead of HTTP-01. Requires `ACME_DNS_PROVIDER` and `ACME_ACCOUNT_EMAIL`. |
| `ACME_DNS_PROVIDER` | `dns_xxx` | DNS provider identifier for the DNS-01 challenge (e.g. `dns_cf` for Cloudflare). See [acme.sh DNS providers](https://github.com/acmesh-official/acme.sh/wiki/dnsapi). |
| `ACME_ACCOUNT_EMAIL` | `me@example.com` | Email address for the Let's Encrypt account. Used for expiry notifications. |
| `ENABLE_SSL_SNI` | `n` | Set to `y` to issue separate certificates per domain, allowing more than 100 domains. Note: some email clients cannot connect using alternative hostnames with SNI. See [Dovecot SNI docs](https://doc.dovecot.org/admin_manual/ssl/sni_support). |
| `ADDITIONAL_SAN` | _(empty)_ | Comma-separated list of additional Subject Alternative Names for the certificate. Supports wildcard expansion over all mail domains (e.g. `imap.*,smtp.*`). |
| `AUTODISCOVER_SAN` | `y` | Obtain certificates for `autodiscover.*` and `autoconfig.*` subdomains of each mail domain. Set to `n` if a reverse proxy already handles those domains. |
| `ADDITIONAL_SERVER_NAMES` | _(empty)_ | Comma-separated alternative hostnames for the mailcow web UI (maps to Nginx `server_name`). Useful when `ADDITIONAL_SAN` includes `mail.*` and you need that name to resolve to the UI. |
| `ONLY_MAILCOW_HOSTNAME` | `n` | Set to `y` to request a certificate only for `MAILCOW_HOSTNAME`, skipping additional SANs. |
| `LE_STAGING` | `n` | Set to `y` to use the Let's Encrypt staging environment. Useful for testing certificate issuance without hitting rate limits. |
| `DIRECTORY_URL` | _(empty)_ | Custom ACME directory URL. Leave empty to use Let's Encrypt production. Set to the staging URL or another ACME-compatible CA. |
| `SKIP_IP_CHECK` | `n` | Set to `y` to skip the IPv4 check in the ACME container. Use with caution. |
| `SKIP_HTTP_VERIFICATION` | `n` | Set to `y` to skip HTTP verification in the ACME container. Use with caution. |
| `SKIP_UNBOUND_HEALTHCHECK` | `n` | Set to `y` to skip Unbound DNS resolver health checks. Not recommended. |

## Optional services

| Variable | Default | Description |
|---|---|---|
| `SKIP_CLAMD` | `n` (auto-set) | Set to `y` to disable ClamAV anti-virus. Rspamd will auto-detect a missing ClamAV container. Automatically set to `y` on systems with ≤2.5 GiB RAM. |
| `SKIP_OLEFY` | `n` | Set to `y` to disable Olefy (Office document anti-virus). Rspamd will auto-detect a missing Olefy container. |
| `SKIP_SOGO` | `n` | Set to `y` to disable SOGo, which disables webmail, DAV protocols, and ActiveSync support. Experimental and unsupported. |
| `SKIP_FTS` | `n` | Set to `y` to disable Dovecot full-text search (FTS). Useful on low-memory or low-threaded systems. |
| `FTS_HEAP` | `128` | Maximum heap size in MB for the Dovecot FTS indexer process (Flatcurve/Xapian backend). Monitor resource usage when tuning. |
| `FTS_PROCS` | `1` | Maximum number of Dovecot indexer worker processes. Too many processes can cause high CPU and disk I/O. See [Dovecot indexer-worker docs](https://doc.dovecot.org/configuration_manual/service_configuration/#indexer-worker). |

## Networking

<Warning>
  Changing `ENABLE_IPV6` requires a full Docker stack rebuild (`docker compose down && docker compose up -d`). Ensure IPv6 is correctly configured on the host before enabling it to avoid creating an open relay.
</Warning>

| Variable | Default | Description |
|---|---|---|
| `IPV4_NETWORK` | `172.22.1` | Internal Docker IPv4 /24 subnet prefix (expands to `n.n.n.0/24`). Use private IPv4 addresses only. |
| `IPV6_NETWORK` | `fd4d:6169:6c63:6f77::/64` | Internal Docker IPv6 subnet in the `fc00::/7` range. Use private IPv6 addresses only. |
| `ENABLE_IPV6` | `true` | Enable IPv6 within mailcow (`true`/`false`). |
| `SNAT_TO_SOURCE` | _(empty)_ | IPv4 address to use for outgoing SNAT connections. Leave empty to use the default route. |
| `SNAT6_TO_SOURCE` | _(empty)_ | IPv6 address to use for outgoing SNAT connections. Leave empty to use the default route. |
| `DISABLE_NETFILTER_ISOLATION_RULE` | `n` | Set to `y` to prevent mailcow from adding an iptables/nftables rule that isolates the Docker network. Disabling this may expose container ports to other hosts on the same subnet. |

## API

| Variable | Default | Description |
|---|---|---|
| `API_KEY` | _(empty)_ | Read-write API key for the web UI API. Allowed characters: `a-z`, `A-Z`, `0-9`, `-`. Must define `API_ALLOW_FROM`. |
| `API_KEY_READ_ONLY` | _(empty)_ | Read-only API key for the web UI API. Same character restrictions as `API_KEY`. |
| `API_ALLOW_FROM` | _(empty)_ | Comma-separated list of IP addresses allowed to use the API (e.g. `172.22.1.1,127.0.0.1`). Required when setting an API key. |

## Watchdog

| Variable | Default | Description |
|---|---|---|
| `USE_WATCHDOG` | `y` | Enable the watchdog to automatically restart unhealthy containers. |
| `WATCHDOG_NOTIFY_EMAIL` | _(empty)_ | Comma-separated list of email addresses for watchdog alert notifications (no quotes, no spaces). Mails are sent unsigned from `watchdog@MAILCOW_HOSTNAME`. Use external recipients and a permissive DMARC policy (`p=none`) for the mailcow hostname. |
| `WATCHDOG_NOTIFY_BAN` | `n` | Set to `y` to send notifications when an IP is banned, including a WHOIS lookup. |
| `WATCHDOG_NOTIFY_WEBHOOK` | _(empty)_ | Webhook URL to receive watchdog notifications as HTTP POST requests with `application/json` content type. Compatible with Discord, Slack, and similar services. |
| `WATCHDOG_NOTIFY_WEBHOOK_BODY` | _(empty)_ | JSON body for the webhook POST request (use single quotes). Available template variables: `${SUBJECT}`, `${BODY}`. Example: `'{"username": "mailcow Watchdog", "content": "**${SUBJECT}**\n${BODY}"}'` |
| `WATCHDOG_SUBJECT` | _(empty)_ | Custom subject line for watchdog alert emails. Defaults to `Watchdog ALERT` followed by the error message. |
| `WATCHDOG_EXTERNAL_CHECKS` | `n` | Set to `y` to check if mailcow is an open relay. Requires a SAL from servercow.de. Opt-in and anonymous; no data is collected. |
| `WATCHDOG_VERBOSE` | `n` | Set to `y` to enable verbose watchdog logging. |
| `WATCHDOG_NOTIFY_START` | `y` | Set to `n` to suppress the notification sent when the watchdog starts. |

## Miscellaneous

| Variable | Default | Description |
|---|---|---|
| `LOG_LINES` | `9999` | Maximum number of log lines per service to keep in Redis. |
| `MAILDIR_SUB` | `Maildir` | Subdirectory name under the mail home directory for storing mail (`mail_home` is `~/Maildir`). |
| `SOGO_EXPIRE_SESSION` | `480` | SOGo session timeout in minutes. |
| `SOGO_URL_ENCRYPTION_KEY` | _(random 16-char)_ | Exactly 16 characters (A-Za-z0-9). Used by SOGo to encrypt email addresses in URLs. Auto-generated on first run. |
| `DOVECOT_MASTER_USER` | _(empty)_ | Optional Dovecot master username. If set, both `DOVECOT_MASTER_USER` and `DOVECOT_MASTER_PASS` must be provided. Expands to `DOVECOT_MASTER_USER@mailcow.local`. Leave empty to auto-generate. |
| `DOVECOT_MASTER_PASS` | _(empty)_ | Optional Dovecot master password. No special characters. Leave empty to auto-generate. |
| `ALLOW_ADMIN_EMAIL_LOGIN` | `n` | Set to `y` to allow admins to log into SOGo as any email user without a password. |
| `WEBAUTHN_ONLY_TRUSTED_VENDORS` | `n` | Set to `y` to restrict WebAuthn/passkey registration to devices from trusted manufacturers. Root certificates must be placed under `data/web/inc/lib/WebAuthn/rootCertificates`. |
| `SPAMHAUS_DQS_KEY` | _(empty)_ | Spamhaus Data Query Service key. Required if your server's ASN is blocked by Spamhaus (e.g. OVH, AWS, Cloudflare). If empty and a blocked ASN is detected, Spamhaus blocklists are disabled entirely. |
